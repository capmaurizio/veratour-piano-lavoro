# Changelog — 2 Luglio 2026 · 08:10

## Sessione di lavoro

### ✅ Fix: openpyxl non installato online
- **Problema**: L'app su [scaypianolavoro.streamlit.app](https://scaypianolavoro.streamlit.app) dava errore `Import openpyxl failed` durante il caricamento dei file Excel.
- **Causa**: Il file `requirements.txt` era assente → Streamlit Cloud non installava `openpyxl`.
- **Soluzione**: Creato il file `requirements.txt` nella root del progetto con le dipendenze base:
  ```
  streamlit
  pandas
  numpy
  openpyxl
  ```
- **Commit**: `fix: aggiungi requirements.txt con openpyxl per Streamlit Cloud`

---

### ✅ Fix: .venv locale corrotto
- **Problema**: Il virtualenv `.venv` puntava a un Python inesistente (`/scay/veratour/.venv/bin/python3`).
- **Soluzione**: Eliminato e ricreato il `.venv` con `python -m venv .venv`, reinstallate tutte le dipendenze tramite pip.
- **Nota**: `.venv` è nel `.gitignore` → non tracciato da git (corretto).

---

### ✅ Fix: anthropic non installato online
- **Problema**: Dopo il fix di openpyxl, l'app mostrava `No module named 'anthropic'` nella sezione Analisi AI / validazione LLM.
- **Causa**: `anthropic` usato in `validation_llm.py` (righe 449 e 711) non era nel `requirements.txt`.
- **Soluzione**: Aggiunto `anthropic` al `requirements.txt`.
- **Commit**: `fix: aggiungi anthropic a requirements.txt per validazione LLM`
